Relatório - Correção do Matching de Variações e GIFs

Arquivo analisado e alterado: popular_variacao.py

Objetivo

Corrigir exclusivamente o matching entre VariacaoExercicio e os GIFs armazenados no Cloudinary, sem alterar models, migrations, estrutura do projeto, relacionamentos, tabelas ou dados existentes fora do campo variacao.gif.

Resumo do Problema

O script anterior fazia uma comparação frágil entre o nome da variação e o nome do GIF. Isso falhava em casos com acentos, underscores, hífens, parênteses, diferenças de caixa, caracteres especiais, sufixos aleatórios do Cloudinary e variações com nomes genéricos como "Máquina" ou "Barra".

O caso principal corrigido foi a necessidade de comparar o nome completo: 

exercicio.nome + variacao.nome

Exemplo:

Crucifixo + Inclinado halter
= Crucifixo Inclinado halter
= media/gifs/Crucifixo_Inclinado_Halter_xpto123

Alterações Realizadas

  • O fluxo passou a processar variação por variação, buscando o melhor GIF correspondente.
  • Foi criada uma montagem explícita do nome completo com exercicio.nome e variacao.nome.
  • Foi criada uma normalização mais robusta de texto.
  • Foi adicionada comparação por tokens com pontuação de similaridade.
  • Foi adicionada paginação na consulta ao Cloudinary usando next_cursor.
  • O salvamento passou a usar public_id no FileField, alinhado ao storage Cloudinary do projeto.
  • Os logs foram ajustados para mostrar o resultado por variação e o total associado.

Normalização Implementada

A função de limpeza passou a tratar:

  • remoção de acentos;
  • conversão para caixa baixa;
  • conversão de underscores, hífens, barras e parênteses para espaços;
  • remoção de caracteres especiais;
  • remoção da extensão do arquivo;
  • compactação de espaços repetidos;
  • remoção de sufixos aleatórios finais do Cloudinary quando parecem hashes, como _xpto123.

Tokens e Equivalências

Para aumentar a chance de match correto sem depender de substring literal, o script agora compara tokens relevantes e remove palavras de baixo valor.

Stopwords usadas:

a, as, com, da, das, de, do, dos, e, em, na, no, para

Equivalências adicionadas:

halteres -> halter
bracos -> braco
cabos -> cabo
ombros -> ombro
gluteos -> gluteo

Regra de Pontuação

Foi adicionada a função pontuar_match. Ela atribui score para:

  • match exato normalizado;
  • match compacto, ignorando espaços;
  • match por cobertura alta de tokens.
Casos fracos continuam sem associação, reduzindo o risco de vincular um GIF errado a uma variação apenas porque compartilha uma palavra genérica.

Salvamento no Campo GIF

Antes o script fazia:

variacao.gif = item["secure_url"]

Agora faz:

variacao.gif.name = public_id
variacao.save(update_fields=["gif"])

O motivo é que o projeto usa MediaCloudinaryStorage como storage padrão, e o comando existente upload_gifs_cloudinary.py também salva public_id. Esse formato é compatível com o uso de variacao.gif.url nos templates.

  • Não alterei migrations.

Observações

Durante a validação, o projeto imprimiu variáveis do Cloudinary por causa de prints existentes em projeto/settings.py. Esses prints não foram alterados, pois estavam fora do escopo solicitado.

Também foram observadas mudanças já existentes em core/models.py e uma migration não rastreada. Esses arquivos foram deixados intocados.
